Exercise 1 - Getting started with OpenGL

Learning objectives The purpose of this set of exercises is to get started with OpenGL. You will:
  • Setup an OpenGL application from scratch;
  • Create a GLFW window and an OpenGL context;
  • Compile and use simple shader programs;
  • Setup the buffers necessary for rendering with OpenGL (Vertex Buffer Object (VBO), Vertex Array Object (VAO) and Element Buffer Object (EBO));
  • Draw simple shapes;
  • Develop a basic understanding of the Normalized Device Coordinates (NDC) and of the window coordinates.
- We will be using the tutorials from learnopengl.com as our main OpenGL development resource.
- A handy OpenGL quick reference card can be downloaded here:
https://www.khronos.org/files/opengl4-quick-reference-card.pdf

Exercise 1.4
Create a disc using multiple triangles.
  • The disc must have a radius of 0.5 and a center in (0,0).
  • Compute the vertex positions using a loop.
  • You can use the math.h sin() and cos() functions to estimate the positions of the vertices in the unit circle.
  • Color the vertices of your own choice.
Exercise 1.5
Rotating cube
  • Create a square
  • Make the square rotate by updating the vertex positions of the posVBO every frame (sin and cos functions are your friends).
  • Make sure to update existing buffers instead of creating new buffers each frame (this could cause a memory leak).
Exercise 1.6
Optimizing geometry part one.
  • Now that you know the basics about constructing and drawing triangle meshes with OpenGL, let's see how we can optimize it for when we have more complicated scenes.
  • Let's change the code such that it only uses a single buffer. This can be done by creating a buffer twice as big as previous:
    • Use the first half for vertex positions;
    • Use the last haft for colors.
  • Now you need glVertexAttribPointer arguments that match the new layout. (stride and offest, see documentation).
Exercise 1.7
Optimizing geometry part two.
  • This should be performed on the result from exercise 1.4
  • An even more optimal layout is to use an interleaved layout.
    • This means that you will still need one buffer to store both the vertex and color information.
    • But the data should be stored using a vertex, then a color, and then the next vertex and the next color etc.
    • Example:
      [vertex0 (x,y,z)][ color0 (r,g,b)][vertex 1 (x,y,z)][color1 (r,g,b)]
    • Now you need glVertexAttribPointer arguments that match the new layout (stride and offest, see documentation).
Exercise 1.8
Creating and indexed mesh of triangles.
  • This should be performed on the result from exercise 1.7.
  • Finally, the shape that we produced has a lot of redundant vertices. For instance, the vertex at the center of the disk is repeated for every triangle.
    • Solve this limitation by defining the vertices only once in a Vertex Buffer Object (VBO), and defining an Element Buffer Object (EBO) with a list of triangle indices.
    • Take the outcome of exercise 1.2 as an example of using a VBO and an EBO.
    • To render with an EBO you should call glDrawElements instead of lgDrawArrays.
Exercise 1.9
Simple drawing program - OPTIONAL CHALLENGE
  • Extend the program to be a simple paint program
    • Change the drawing mode to GL_POINTS.
    • Insert the statement glPointSize(4.0f); into the main function.
    • When the user clicks or draws on the canvas the vertex buffer should be updated with the position (transformed into normalized device coordinates).
    • Use the mouse state (glfwGetMouseButton (GLFWwindow *window, int button) and glfwGetCursorPos (GLFWwindow *window, double *xpos, double *ypos)) on the canvas to capture mouse drags.
    • Remember to change coordinates from the window coordinates to the normalized device coordinates (xndc = xpos/width * 2 - 1).
